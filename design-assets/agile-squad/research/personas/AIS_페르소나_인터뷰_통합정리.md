# AIS 유료 체험 방식 — 페르소나 인터뷰 통합 정리

> 인터뷰 일자: 2026-06-29  
> 대상: 타겟 A 조기원 (현업 활용·커뮤니티 검증형) / 타겟 B 서담채 (청취·트렌드 스캔형)  
> 목적: 유료 기능 체험 방식(횟수 제한 vs 기간 제한) 및 적정 수치 도출

---

## 1. 체험 방식 선호

| 항목 | 조기원 (타겟 A) | 서담채 (타겟 B) |
|---|---|---|
| 선호 방식 | **B방식 (기간 제한)** | **B방식 (기간 제한)** |
| 이유 | 횟수 제한은 쓸 때마다 '아껴야 하나' 심리가 생겨 기능 탐색이 방해됨 | 횟수 신경 쓰면 기능을 제대로 못 써봄. 일상에 자연스럽게 녹아드는 경험이 중요 |

**→ 두 페르소나 모두 B방식(기간 제한) 선호. 이유는 다르지만 결론 동일.**

---

## 2. 적정 수치 도출

### 2-1. A방식 (횟수 제한) — 참고 수치

| | 조기원 (타겟 A) | 서담채 (타겟 B) |
|---|---|---|
| 판단 가능 최소 횟수 | **최소 20회** (15회는 부족) | **5~7회** |
| 근거 | 일주일 수치 확인 필요 기사 3~4건, 한 달 15회는 익숙해질 만하면 소진 | 첫 1~2회 탐색, 3~4회부터 자기 방식 활용 → 유불리 판단 가능 |

> 두 페르소나 간 적정 횟수 차이가 큼(20회 vs 5~7회). A방식 채택 시 타겟별 한도를 달리해야 하는 복잡성 발생.

### 2-2. B방식 (기간 제한) — 핵심 수치

| | 조기원 (타겟 A) | 서담채 (타겟 B) |
|---|---|---|
| 초기 답변 | 2주 | 7일 |
| 수정/최종 답변 | **2주** | **10일~2주** |
| 근거 | 첫 3~4일은 기능 익히기, 이후 실무 적용. 월말 결산 등 바쁜 타이밍이 1회라도 걸려야 가치 체감 | 7일 중 기능을 제대로 쓰는 날 4~5일, 유료 기능 진입은 3회 될까 말까 → 7일은 현실적으로 부족 |

**→ 두 페르소나 모두 14일(2주)로 수렴.**

### 2-3. 확신 형성 최소 횟수 (기능 가치 체감 기준)

| | 조기원 (타겟 A) | 서담채 (타겟 B) |
|---|---|---|
| 최소 횟수 | **5회** | **5회** |
| 근거 | 처음 2~3회 신기함, 4~5회째 손에 익고 없을 때 빈자리 느껴짐 | 처음 2~3회 신기함, 4~5회부터 없을 때 빈자리 느껴짐 |

**→ 두 페르소나 공통. 5회가 '습관 형성 임계점'.**

## 3. 전환 트리거 (결정적 구독 버튼 클릭 순간)

| | 조기원 (타겟 A) | 서담채 (타겟 B) |
|---|---|---|
| 트리거 맥락 | 업무 시간 절감 — ECOS 헤매던 걸 팝업 하나로 해결, CSV 뽑아 설명자료 작성 시간 단축 | 취준 실용성 — 자소서 작성 중 산업 트렌드 요약, 면접 준비 중 브랜드 이슈 파악 |
| 핵심 감각 | "시간 아껴준 게 눈에 보여야 지갑이 열린다" | "이거 없으면 나 진짜 불편하겠다" |

## 4. 이탈 리스크

| | 조기원 (타겟 A) | 서담채 (타겟 B) |
|---|---|---|
| 주요 이탈 원인 | 타이밍 미스 — 체험 기간 중 업무 바쁜 시기와 겹치지 않으면 그냥 흘려보냄 | 비용 민감도 (취준 중, 고정 지출 예민). "좋긴 한데 지금 당장 필요한가?"에서 걸림 |
| 공통 리스크 | 체험 기간에 기능을 제대로 써보지 못한 경우 "나한테는 필요 없다"로 귀결 |

## 5. 체험 종료 안내 선호 방식

두 페르소나 모두 **사용 기록 시각화**가 핵심 전환 장치로 작동.

| 항목 | 조기원 (타겟 A) | 서담채 (타겟 B) |
|---|---|---|
| 선호 안내 | "지난 2주간 N회 사용, 주로 이런 기사에서 활용" 사용 요약 | 종료 2~3일 전 "이번 주 활용 요약" 리포트 + 첫 달 할인 |
| 핵심 심리 | 내 데이터가 근거가 되는 것 → 할인보다 설득력 높음 | "내가 이만큼 썼구나" → 아깝다는 감각 유발 |

**공통 권장 포맷:** 기간 종료 D-3 알림 + 실사용 요약 리포트 (사용 횟수, 활용 기사 유형, 절감 시간 추정) + 첫 달 할인 옵션

## 6. 잔여 횟수 알림 반응 (A방식 채택 시 참고)

| | 조기원 (타겟 A) | 서담채 (타겟 B) |
|---|---|---|
| "3회 남았어요" 알림 반응 | 서두르거나 아끼거나 양가적 반응 | 남은 거 다 써버리는 타입 ("남기면 손해" 심리) |
| 리스크 | 아끼다가 오히려 기능 탐색 못 함 | 막 눌러보다가 피상적 사용으로 끝남 |

## 7. 기능 인지도 격차

두 페르소나 공통으로 동일한 패턴.

| 기능 | 인지도 | 비고 |
|---|---|---|
| 에픽 AI | 상 | "기사 보면서 바로 쓴다"는 게 직관적으로 이해됨 |
| 프리미엄9 | 중 | "9가지 뭐가 있다는 건지" 체험 전 감이 안 옴. 큐레이션 기준 설명 필요 |
| 한경에이셀 | 하 | 이름에서 기능이 전혀 연상 안 됨. 서담채: "Excel이에요? 고유명사예요?", 조기원: "기관 투자자용 느낌" → 클릭 동기 자체가 없음 |

**→ 한경에이셀 온보딩 copy 개선이 가장 시급.**

## 8. 종합 결론 및 설계 권고

### 체험 방식 결정

| 항목 | 권고안 |
|---|---|
| 방식 | **B방식 — 기간 제한 무료 체험** |
| 기간 | **14일 (2주)** |
| 근거 | 두 페르소나 모두 수렴. 기능 습관 형성 임계점(5회)을 2주 안에 자연스럽게 충족 가능 |

### 이탈 방지 설계 권고

- **온보딩 시 사용 시나리오 제시**: 타겟A(조기원)는 업무 맥락(보고서 인용·검증), 타겟B(서담채)는 취준 맥락(자소서·면접 준비) — 진입 시점에 "이럴 때 써보세요" 가이드 제공
- **중간 사용 독려 알림**: D+7 시점에 "아직 에픽 AI를 써보지 않으셨어요" 등 넛지 알림 (타이밍 미스 방지)
- **종료 D-3 리포트**: 실사용 요약 + 첫 달 할인으로 자연스러운 전환 유도

### 추가 검토 필요 사항

- 한경에이셀·프리미엄9 온보딩 copy 전면 개선 (기능명만으로 가치 전달 안 됨)
- 타겟B(서담채/취준생) 대상 가격 민감도 고려 — 학생/취준 플랜 또는 낮은 진입 가격 옵션 검토
- 14일 체험 vs 현재 PRD의 "일 10회 무료 한도" 정책 간 정합성 재검토 필요
